Each entry must consist of a name and a value, separated by a space. The name can have special meanings:
  • Names beginning with an underscore "_" cause the entry to use the Custom Command format, so it supports replacements, but also e.g. requires escaping of special characters $ and \ when intended to be used literally. Generally the $1- replacement will contain the text given as a parameter when this Preset is used.
  • Entries with other names are interpreted as plain text.
  • Entries where the name begins with "#" are entirely ignored.
  • An entry with the name _global_<type> (where <type> can be highlight, highlightBlacklist, ignore, ignoreBlacklist, filter, msgcolor, usercolor, badge or notification) will automatically be applied to settings of that type (or at least the part of it that uses Highlight-type matching) as if it was prefixed with ccf:_global_<type>| (see help). Use _global to apply to all of these (although specific type takes precedence if present).

These entries can be used in pretty much every setting that uses the Highlight format, not just the Highlight list, however note that a restart may be required for changes to apply to settings that are not the Highlight, Ignore or Filter list.

Read the Presets help for how to use the entries on this list.
